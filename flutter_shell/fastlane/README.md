# Fastlane — AppForge shell

Generated lanes. **Do not edit this file** — the source of truth is
`Fastfile`. See `docs/FASTLANE.md` (repo root) for setup and the full
walk-through.

## Available lanes

### Android
| Lane | What it does |
|---|---|
| `android build` | Local AAB only (no upload). |
| `android beta` | Build AAB → upload to Play Store **Internal Testing**. |
| `android promote rollout:0.1` | Promote latest internal build → **Production** at 10% staged rollout. |

### iOS
| Lane | What it does |
|---|---|
| `ios build` | Local IPA only (no upload). Requires Xcode + signing config. |
| `ios beta` | Build IPA → upload to **TestFlight**. |
| `ios promote` | Submit latest TestFlight build for **App Store** review. |

### Utility
| Lane | What it does |
|---|---|
| `doctor` | Verify credentials are present and paths resolve. |

## Don't run `fastlane` directly

Always go through `scripts/fastlane.sh <app> <platform> <lane>` from
the repo root. The wrapper:
- reads per-app values from `apps/<app>/app.yaml` and `apps/<app>/overrides/fastlane/Appfile`
- exports the env vars the Fastfile expects
- `cd`s into the per-app build directory
- runs `bundle exec fastlane <platform> <lane>`
